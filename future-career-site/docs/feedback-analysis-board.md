# 第一批用户反馈分析看板

用于整理 10-20 位早期测试用户的反馈。目标不是追求所有人都说准确，而是找出下一轮最值得修的地方。

## 1. 核心指标

| 指标 | 目标 | 当前记录 |
| --- | --- | --- |
| 完成测试人数 | 10-20 人 | 待填 |
| 平均推荐准确度 | >= 4.0 / 5 | 待填 |
| 平均解释清晰度 | >= 4.0 / 5 | 待填 |
| 平均城市适配度 | >= 3.5 / 5 | 待填 |
| 平均专业适配度 | >= 3.5 / 5 | 待填 |
| 愿意做入门项目比例 | >= 50% | 待填 |
| 点开详情比例 | >= 60% | 待填 |

## 2. 高频准确推荐

记录被多位用户认为“比较像我”的职业。

| 职业 | 出现次数 | 准确原因 | 是否值得强化 |
| --- | --- | --- | --- |
| 待填 | 待填 | 待填 | 待填 |

## 3. 高频不准原因

| 问题类型 | 典型反馈 | 可能原因 | 下一步动作 |
| --- | --- | --- | --- |
| 职业推荐不准 | 用户原本更偏创意/稳定/技术 | 题目权重或职业库覆盖不足 | 调整权重或补职业 |
| 城市推荐不准 | 城市离家太远/成本太高 | 城市数据缺少成本、家庭约束 | 增加城市约束解释 |
| 专业推荐太宽 | 用户不知道该主攻哪个专业 | 专业等级解释不足 | 强化主攻/辅修/补充差异 |
| 解释不可信 | 用户想知道依据 | 数据来源等级不足 | 补公开资料来源 |
| 页面操作困难 | 手机按钮太多/文字挤 | 移动端布局不足 | 优化移动端组件 |

## 4. 下一轮迭代优先级

使用 `high / medium / low` 标记。

| 优先级 | 任务 | 判断依据 | 完成标准 |
| --- | --- | --- | --- |
| high | 修复最常见不准职业 | 3 人以上提到同类问题 | 推荐解释或权重更新 |
| high | 补城市成本与离家解释 | 城市适配度低于 3.5 | 城市详情增加约束说明 |
| medium | 精修专业推荐层级 | 专业适配度低于 3.5 | 主攻/辅修/补充更清晰 |
| medium | 补真实岗位关键词 | 用户质疑可信度 | 每个高频职业有岗位关键词 |
| low | 页面文案润色 | 单个用户提出 | 不影响主流程 |

## 5. 复盘问题

每收集 10 份反馈后回答一次：

1. 哪 3 个职业最常被认为准确？
2. 哪 3 个职业最常被认为不准确？
3. 哪个城市推荐争议最大？
4. 用户最想增加什么信息：薪资、大学专业、岗位、城市成本，还是真实案例？
5. 有没有某一道题明显让用户困惑？
6. 如果只能改 3 件事，应该改哪 3 件？

## 6. 数据文件

逐条反馈记录在：

```text
data/user-testing-log.sample.csv
```

建议正式收集时复制一份：

```text
data/user-testing-log.private.csv
```

`private.csv` 不应上传到公开仓库，避免暴露测试者个人信息。
