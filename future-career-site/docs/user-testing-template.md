# 小规模用户测试模板

第一阶段完成后，建议找 20-50 位真实用户测试。目标不是证明网站完美，而是发现推荐哪里不准、哪里解释不清、哪里让用户想继续探索。

## 测试对象

建议覆盖：

- 高中生：正在考虑大学专业。
- 大学生：正在考虑实习、转专业或未来职业。
- 研究生或求职者：正在比较职业方向。
- 家长或老师：从指导视角看报告是否可信。

## 测试流程

1. 让用户独立完成测试，不提前解释推荐逻辑。
2. 记录用户完成时间。
3. 让用户查看第 1 推荐职业、职业对比、城市详情和 7 天行动计划。
4. 让用户用 1-5 分评价推荐准确度和解释清晰度。
5. 追问：哪一个推荐最有启发？哪一个最不准？为什么？
6. 让用户选择是否愿意真的做 7 天行动计划。

## 推荐准确度问题

可以直接问：

1. 第 1 推荐职业是否像你？
2. 前 3 个职业是否覆盖了你真正感兴趣的方向？
3. 推荐城市是否符合你的生活和家庭约束？
4. 推荐专业是否看起来可行？
5. 你是否理解为什么系统这么推荐？

## 记录指标

核心指标：

- 推荐准确度：1-5 分。
- 解释清晰度：1-5 分。
- 行动意愿：是否愿意做 7 天计划。
- 最不准原因：职业不感兴趣、地点不合适、门槛判断不准、专业不合适、解释不可信。
- 用户原本目标和系统推荐是否一致。

## 判断第一阶段是否通过

建议目标：

- 70% 以上用户认为前 3 个职业“至少有一个值得认真了解”。
- 60% 以上用户能说清楚为什么系统推荐第 1 职业。
- 50% 以上用户愿意尝试一个 7 天行动任务。
- 高频“不准原因”可以被归类，并能对应到题目、职业权重或城市数据的调整。

## 下一轮迭代

测试后优先处理：

1. 被频繁认为“不准”的职业。
2. 用户看不懂的解释文本。
3. 明显缺失的职业方向。
4. 城市推荐和家庭/成本约束冲突的情况。
5. 专业推荐过宽或过窄的问题。
